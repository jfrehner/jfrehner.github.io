Thoughts and ideas on how to organize color in a webapp

published: 10.6.2021, 10 minute read

The Problem

I always asked myself how different colors within a webapp could be organized so that it makes sense and is not overwhelming. In the past, I did not think much about it as my projects always had already a pretty much defined color palette and organized it neatly within various scss-variables.

But when I encountered a project where we had to build up the app from scratch, I noticed that I didn't like to define the colors "on the fly". "Oh we need a color for the menu-bar - define one in the menu-bar.scss", "The footer needs a color? Declare it in the footer.scss" and so on. This leads, consequently, to a beautiful mess - colors defined everywhere, maybe even the same shades that look like they are the same color, but might be off for a little bit etc.

I saw the problem: colors not defined and used consistently, but I did not know how to solve it. I knew there had to be a systematic approach on how one can organize color. So this post shall outline my ideas and thoughts about how I could define, organize and use color in webapps. A word of caution: this is my personal opinion, work in progress and very much subject to change :)

Thoughts

The following is a collection of knowledge acquired from various articles available online (see at the end for a reference).

Color types, color wheel, color palettes
  • The color wheel defines different colors
    • primary - yellow, red, blue
    • secondary - mixes of primary colors e.g. orange, green
    • tertiary - mix of a primary and secondary color e.g. yellow-green
  • A color palette is a set of colors that work well together.
  • There are different types of color palettes (all based on the color wheel):
    • Analogous
    • Complementary
    • Split complementary
    • Double complementary (tetradic)
    • Triadic
    • Monochromatic
Create and use a color palette
  • Rather use fewer colors - it is easier to create visual hierarchy and contrast with ca. 3 - 5 colors than with more
  • Obviously: follow the color guidelines of your brand if there are any
  • Choose a color palette type to start with and iterate - change colors, add hues etc. in subsequent iterations if necessary
  • 60 - 30 - 10 Rule: The dominant color should be used in 60% of cases, the secondary color in 30% and the accent color in around 10% of cases. Reason: this creates a nice balance and prevents a too colorful design
  • Pay attention to consistency: if red is used for error-messages, use it consistently for error-messages and not suddenly for a link etc.
  • Accent colors should only be used for things that should stand out
  • Don't use color alone to convey information - color blind people might not be able to perceive it correctly
Visual design

This is not strictly only about color, but color plays obviously an important part in visual design.

Kelly Gordon (2020) mentions five principles

  1. Use relative size/scale to show importance and rank
    The most important elements need to be the biggest. Gordon suggests using no more than 3 different sizes.
  2. Provide consistent visual hierarchy
    Guide the eye through the page so that it becomes clear what the importance of specific design elements are. Can be done through size, space, placement, colors etc.
  3. Provide visual balance (using scale, color etc.) along an imaginary axis
  4. Use contrast (e.g. in size, color etc.) to signify that elemetns differ
  5. Use the gestalt principles (e.g. similarity, continuation, proximity etc.)
